## Pricing Rules

### Minimums

- **Minimum project fee: $500.** Below this, the admin overhead
  (agreement, communication, invoicing) eats your margin. Politely
  decline or suggest a larger scope.
- **Minimum hourly rate: $75/hr.** This is your floor, not your
  target. Raise it as you build reputation.

### Fixed Price vs Hourly

| Use Fixed Price When...            | Use Hourly When...                   |
| ---------------------------------- | ------------------------------------ |
| Scope is clear and bounded         | Scope is exploratory or evolving     |
| You can estimate hours confidently | Client wants ongoing support         |
| Project is under 40 hours          | Project is open-ended                |
| Client wants predictability        | You're unsure how deep the issues go |

**Fixed price rule:** Estimate your hours honestly, then multiply by
1.5x. That's your quote. The extra 0.5x covers the surprises you
can't predict yet. If you finish early, great â€” you earned efficiency.
If it takes longer, you're covered.

### The Upcharge Triggers

Add 25-50% to your base rate when:

- [ ] Client needs it "ASAP" or "by Monday" (rush fee)
- [ ] Codebase has no tests and no documentation (archaeology fee)
- [ ] Client can't clearly explain what the app does (discovery fee)
- [ ] Multiple languages or frameworks involved (complexity fee)
- [ ] You'll need access to production systems (responsibility fee)

You don't have to itemize these on the invoice. Just factor them
into the quote.

## Scope Creep Defense

### The Golden Rule

**If the scope changes, the price changes.**

Say it out loud before every client call. Write it in every proposal.

### When They Ask for "Just One More Thing"

Use this exact script:

> "Happy to help with that! That's outside our current scope, so let
> me put together a quick quote for the additional work. I'll have it
> to you by [tomorrow/end of day]."

Do NOT:

- Say "sure, I can squeeze that in"
- Say "no problem, it's small"
- Do it and mention it later
- Feel guilty about it

The scope was agreed on. Additional work = additional pay. This is
normal. This is professional. This is how it works.

### The 10% Rule

If a "small addition" would take less than 10% of the original
project hours AND you're feeling generous, you can absorb it as
goodwill. But ONLY if:

- The main project is going well
- The client has been respectful and communicative
- You genuinely want to do it (not guilt)
- It will not delay the original deliverables

If any of those aren't true, quote it separately.

## The ADHD-Specific Rules

These are for YOU, specifically, based on what you told me about
how your brain works.

### 1. Sleep on every proposal.

Write the quote. Save the draft. Send it TOMORROW. The overnight
gap catches the "yes impulse" before it costs you.

### 2. One active project at a time.

You can have multiple clients in different phases (one in
assessment, one in progress, one in warranty). But only ONE should
be in active "building" mode at a time. Your brain does not
multithread on deep work. Don't pretend it does.

### 3. Time-box the free assessment.

Set a 30-minute timer. When it rings, STOP. Write up what you found.
Send it. The assessment is a gift â€” don't let it become the project.

### 4. Track hours from minute one.

Use a timer app (Toggl is free). Start it when you open their repo.
Stop it when you close it. Do NOT estimate hours from memory at the
end of the week. Your sense of time is unreliable (this is an ADHD
thing, not a you thing).

### 5. Celebrate finishing.

When you deliver a project, do something that feels good. Coffee
shop trip. Favorite meal. A walk. Your brain needs the dopamine hit
of completion to build the habit loop. Don't just immediately open
the next project.

## Quick Reference: Project Sizes

| Size       | Hours     | Typical Fee    | Examples                                      |
| ---------- | --------- | -------------- | --------------------------------------------- |
| **Small**  | 5-15 hrs  | $500-$1,500    | Security audit report, bug fixes, code review |
| **Medium** | 15-40 hrs | $1,500-$4,000  | Auth refactor, API hardening, test suite      |
| **Large**  | 40-80 hrs | $4,000-$10,000 | Full codebase refactor, production hardening  |
| **XL**     | 80+ hrs   | $10,000+       | Architecture overhaul, multi-system work      |

For Large and XL: break into phases with separate quotes per phase.
Never quote 80+ hours as a single deliverable. Break it up.
